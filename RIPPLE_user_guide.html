RIPPLE

User Guide — Apply to Your Own Data

A step-by-step recipe for detecting distance-dependent gene expression gradients from any query cell type, on any spatial transcriptomics platform.

When to Use RIPPLE

RIPPLE answers a specific question. Make sure it's yours.

Use RIPPLE when your question is:

"Which genes in cell type B change expression as a function of physical distance from cell type A?"

Good fit

  • Paracrine signaling from an inflammatory/secretory population
  • Tumor border effects on immune cells
  • Niche signals from stromal cells to stem cells
  • Any "dose-response" spatial question

Not the right tool

  • Finding spatially variable genes with no reference cell type → use SpatialDE/SPARK
  • Cell-cell communication without spatial data → use CellChat/NicheNet
  • Comparing conditions without spatial context → use pseudobulk DE
  • Spatial domain segmentation → use BANKSY/SPACEL

Requirements

  • Spatial transcriptomics data with single-cell or subcellular resolution (Xenium, MERFISH, CosMx, seqFISH, CODEX). Works with spot-based data (Visium) if deconvolution provides cell-type calls.
  • Cell type annotations including your query population (the "source" cell type whose influence you want to measure).
  • Biological replicates — minimum 3, ideally 4+. This is what makes RIPPLE's meta-analysis meaningful. With only 1 sample, you lose the core advantage.
  • Raw count matrix (not normalized). The Poisson GLM requires integer counts.

Input Data Structure

What RIPPLE expects and how to prepare it.

RIPPLE operates on a standard single-cell spatial object. You need four pieces of information per cell:

FieldDescriptionExample
countsRaw integer transcript counts (genes × cells)Seurat RNA@counts or AnnData .X (raw layer)
x, ySpatial coordinates in physical units (μm)Seurat @images$fov@boundaries$centroids
cell_typeCell type annotation including your query type"Macrophage", "Tumor", "Treg", etc.
sample_idBiological replicate identifier"mouse_1", "patient_A"
Total counts are computed, not supplied. RIPPLE calculates total_counts = colSums(counts_matrix) per cell. This serves as the cell-size proxy for the Poisson offset. Don't use library-size-normalized data — use the raw counts layer.

Terminology

Query = the cell type whose spatial influence you want to measure (e.g., inflammatory myeloid cells, tumor cells, CAFs). Target = any other cell type where you measure gene expression as a function of distance to the nearest query cell. RIPPLE loops over all target cell types automatically.

Parameters & Defaults

What to set, what to leave alone, and when to change defaults.

ParameterDefaultWhat it controlsWhen to change
QUERY_TYPE required The cell type label for the "source" population Always set to your query cell type
MAX_DISTANCE_UM 200 Maximum distance to consider (μm). Cells beyond this are excluded. Reduce for contact-dependent signals (~50); increase for long-range morphogens (~500)
K_NEIGHBORS 1 Number of nearest query cells for distance calculation Set to 3–5 for robustness if query cells are very sparse or segmentation is noisy
MIN_EXPR_CELLS 25 Minimum non-zero cells per sample for a gene to be tested Lower for rare cell types; raise for abundant types to reduce testing burden
MIN_EXPR_PCT 0.01 Minimum fraction of cells expressing a gene (alternative to MIN_EXPR_CELLS) The actual threshold is max(MIN_EXPR_PCT * n_cells, MIN_EXPR_CELLS)
N_PERM 500 Number of permutations for null distribution Increase to 1000+ for publishable permutation p-values (finer resolution)
FDR_THRESHOLD 0.05 Fisher FDR cutoff for significance Standard choice; use 0.1 for discovery, 0.01 for high-confidence sets
SIGN_CONSISTENCY_THRESHOLD 1.0 Required fraction of samples agreeing on β direction Relax to 0.75 if you have many samples (N≥6) and want more power
How to choose MAX_DISTANCE_UM: Think about the biology. Cytokines like IL-1β and TNF-α act at 20–100μm. Growth factors like VEGF diffuse 100–200μm. Morphogens (Wnt, Shh) can act at 200–500μm. Start with 200μm as a sensible default, then check whether your strongest hits show decay within the first 50–100μm (suggesting shorter-range biology) or continue beyond (suggesting longer range).

Stage 1: Per-Sample Poisson GLM

The core statistical model, explained for implementation.

Step 1a: Compute distances

For every cell of a given target cell type, calculate the Euclidean distance to its nearest query cell (within the same sample).

R # For each sample independently: query_coords <- cbind(metadata[cell_type == QUERY_TYPE, x], metadata[cell_type == QUERY_TYPE, y]) target_coords <- cbind(metadata[cell_type == current_target, x], metadata[cell_type == current_target, y]) # k-NN search (k=1 for default; k>1 averages distances) nn <- RANN::nn2(query_coords, target_coords, k = K_NEIGHBORS) if (K_NEIGHBORS == 1) { distances <- as.vector(nn$nn.dists) } else { distances <- rowMeans(nn$nn.dists) } # Cap at MAX_DISTANCE_UM distances[distances > MAX_DISTANCE_UM] <- NA # exclude cells beyond range
Python from sklearn.neighbors import NearestNeighbors import numpy as np # For each sample independently: query_xy = adata[adata.obs["cell_type"] == QUERY_TYPE].obsm["spatial"] target_xy = adata[adata.obs["cell_type"] == current_target].obsm["spatial"] nn = NearestNeighbors(n_neighbors=K_NEIGHBORS, metric="euclidean").fit(query_xy) dists, _ = nn.kneighbors(target_xy) distances = dists.mean(axis=1) if K_NEIGHBORS > 1 else dists[:, 0] # Cap distances[distances > MAX_DISTANCE_UM] = np.nan

Step 1b: Fit Poisson GLM per gene, per sample

For each gene that passes the expression threshold:

The RIPPLE model log(E[Yi])  =  α  +  β · di  +  log(Ti)
R fit_ripple_glm <- function(counts, distances, total_counts) { # Filter valid cells valid <- !is.na(distances) & total_counts > 0 y <- counts[valid] d <- distances[valid] lT <- log(total_counts[valid]) # Check minimum expression if (sum(y > 0) < MIN_EXPR_CELLS) return(NULL) # Fit Poisson GLM with offset fit <- glm(y ~ d + offset(lT), family = poisson()) cs <- summary(fit)$coefficients list( beta = cs["d", "Estimate"], se = cs["d", "Std. Error"], pval = cs["d", "Pr(>|z|)"], dispersion = fit$deviance / fit$df.residual ) }
Python import statsmodels.api as sm def fit_ripple_glm(counts, distances, total_counts, min_expr=25): """Fit Poisson GLM for one gene in one sample.""" valid = ~np.isnan(distances) & (total_counts > 0) y = counts[valid] d = distances[valid] offset = np.log(total_counts[valid]) if (y > 0).sum() < min_expr: return None X = sm.add_constant(d) model = sm.GLM(y, X, family=sm.families.Poisson(), offset=offset) result = model.fit() return { "beta": result.params[1], "se": result.bse[1], "pval": result.pvalues[1], "dispersion": result.deviance / result.df_resid }
Use raw counts, not normalized expression. The Poisson GLM expects integer counts. The offset(log(total_counts)) handles the normalization internally by converting to rates. If you feed in normalized data, the offset correction is meaningless and results will be unreliable.

Combining Across Replicates

Fisher's combined p-value with a sign consistency gate.

After fitting the GLM in each sample, you have one β and one p-value per replicate. To combine them:

Algorithm

R combine_fisher <- function(betas, pvals) { k <- length(pvals) # 1. Sign consistency gate: ALL replicates must agree n_pos <- sum(betas > 0) n_neg <- sum(betas < 0) sign_consistent <- (n_pos == k) | (n_neg == k) if (!sign_consistent) { return(list(fisher_pval = 1, median_beta = median(betas))) } # 2. Fisher's method: X^2 = -2 * sum(log(p_i)) clamped <- pmax(pvals, 1e-15) # avoid log(0) fisher_stat <- -2 * sum(log(clamped)) fisher_pval <- pchisq(fisher_stat, df = 2 * k, lower.tail = FALSE) list(fisher_pval = fisher_pval, median_beta = median(betas)) }
Python from scipy import stats def combine_fisher(betas, pvals): k = len(pvals) # Sign consistency gate n_pos = sum(b > 0 for b in betas) n_neg = sum(b < 0 for b in betas) if n_pos != k and n_neg != k: return {"fisher_pval": 1.0, "median_beta": np.median(betas)} # Fisher's combined test clamped = np.clip(pvals, 1e-15, 1.0) chi2 = -2 * np.sum(np.log(clamped)) fisher_pval = stats.chi2.sf(chi2, df=2 * k) return {"fisher_pval": fisher_pval, "median_beta": np.median(betas)}

Then apply FDR correction

R # Per target cell type results$fisher_fdr <- p.adjust(results$fisher_pval, method = "BH")
Why not just pool all cells and fit one GLM? Because cells from the same mouse are not independent. Pooling creates pseudoreplication — your effective sample size is N=4 mice, not 100,000 cells. By fitting per-mouse and combining, RIPPLE respects the experimental design. This is the single most important methodological choice.
Alternative: REML random-effects meta-analysis. If you want precision-weighted effect sizes (giving more weight to high-powered samples), use meta::metagen(..., method.tau = "REML"). This is useful for effect estimation but can be dominated by one large sample. We recommend Fisher's method for significance calls and REML for effect-size reporting.

Permutation Validation

Confirming that gradients are specific to your query cell type.

A significant gradient could arise simply because query cells are spatially clustered (e.g., in the medulla). The permutation test asks: "Would a random spatially-clustered cell population produce the same gradient?"

Algorithm

  1. For each replicate s, count how many query cells it has: ns
  2. For b = 1 to N_PERM (e.g., 500):
    • In each replicate, randomly sample ns cells from all cell types as "pseudo-query"
    • Recalculate distances from all target cells to nearest pseudo-query cell
    • Fit Poisson GLM with offset → extract βs(b)
    • Pool across replicates using inverse-variance weighting → βcombined(b)
  3. Permutation p-value = (#{|β(b)| ≥ |βobs|} + 1) / (B + 1)
Computational cost. Permutations are the bottleneck. For 500 permutations × 200 genes × 14 cell types, expect several hours on CPU. Consider GPU acceleration for the k-NN distance calculations (the most expensive step), or limit permutations to the top 100–200 genes by effect size.
Not strictly required. The Fisher's combined p-value with sign consistency gate is the primary significance metric. Permutation tests provide orthogonal validation that the signal is query-cell-specific. If compute is limited, skip permutations and rely on Stage 2's confounder control instead.

Stage 2: Confounder Control

Separating your query-specific signal from shared niche effects.

If your query cell type co-localizes with another cell type (e.g., both in the medulla, both at the tumor border), some gradients may reflect the shared niche rather than your query cell's influence. Stage 2 adds the confounder's distance as a second predictor:

Stage 2 Bivariate Model log(E[Yi]) = α + βquery · dquery + βconfound · dconfound + log(Ti)
R # Stage 2: add confounder distance fit2 <- glm(counts ~ dist_to_query + dist_to_confounder + offset(log(total_counts)), family = poisson()) # beta_query now represents the QUERY-SPECIFIC gradient, # after controlling for confounder proximity

Classification

Compare Stage 1 result (significant?) with Stage 2 result (βquery still significant after adding confounder?):

SigSig, strongerenhancedConfounder was suppressing the signal
SigNot sig, same directionunderpoweredLikely real but lost power from collinearity
SigNot sig or reversedniche_drivenExplained by shared niche structure
Choosing your confounder cell type: Pick a cell type that (1) co-localizes with your query cells and (2) is biologically distinct. Examples:
  • Query = inflammatory myeloid → Confounder = monocytes (shared niche, different function)
  • Query = tumor cells → Confounder = CAFs (both in tumor core)
  • Query = Tregs → Confounder = CD4 T cells (shared T cell zones)
If no obvious confounder exists, skip Stage 2 and rely on permutation testing instead.

Interpreting Results

What the numbers mean biologically.

The coefficient β

β valueDirectionBiological interpretation
β < 0Expression higher near queryInduced — query cells upregulate this gene in nearby target cells (paracrine activation)
β > 0Expression lower near queryRepressed — query cells suppress this gene in nearby target cells (paracrine inhibition)
β ≈ 0No spatial relationshipGene expression is independent of proximity to query cells

Magnitude: what does β = −0.003 mean?

The coefficient is in units of log(rate change) per μm. To convert to a rate ratio over a distance of Δd μm:

Rate ratio RR = eβ × Δd

Example: β = −0.003, Δd = 50μm → RR = e−0.15 ≈ 0.86. Cells 50μm away express the gene at 86% the rate of cells adjacent to query cells. A 14% reduction across 50μm.

Adapting to Your Setting

Platform-specific considerations and common modifications.

Xenium / MERFISH / CosMx

Subcellular resolution. RIPPLE works out of the box. The cell-size offset is important because segmentation quality varies cell-to-cell. Use total_counts from the raw count matrix.

Visium / Slide-seq

Spot-based. Requires deconvolution (e.g., cell2location, RCTD) to assign cell types per spot. Use deconvolved cell type proportions as weights or restrict to spots dominated (>70%) by one cell type. Coordinates are spot centroids.

CODEX / Imaging mass cytometry

Protein, not transcript. The Poisson GLM still works if counts are integer (number of antibody detections). For continuous intensity data, consider a Gaussian GLM instead. The offset should be total protein signal per cell.

Stereo-seq / Slide-seq V2

Very high resolution, very sparse per spot. Bin spots to cell-level aggregates before applying RIPPLE. Alternatively, use bin-level counts with a spot-size offset.

Multiple query cell types

To test multiple query populations (e.g., "Are Tregs and M2 macrophages producing similar gradients?"), run RIPPLE separately for each query type. Compare the resulting β values across analyses:

  • Genes significant for both query types → shared niche effect (validate with Stage 2)
  • Genes significant for only one → query-type-specific signal
  • Correlate β vectors across query types to quantify overlap

Bidirectional analysis

RIPPLE natively tests query → target (e.g., how do genes in T cells change near HyMy?). For the reverse — target → query (how do genes in HyMy change near T cells?) — simply swap the query and target labels and re-run. This reveals reciprocal signaling.

Common Pitfalls

Mistakes we made so you don't have to.

1. Using normalized data

The most common mistake. If you feed in log-normalized or SCTransform-normalized data, the offset correction is meaningless (you're "normalizing twice"). Always use the raw integer count matrix.

2. Treating all cells as independent

If you skip the per-sample fitting and just run one GLM on all cells, you have pseudoreplication. With 100,000 cells, nearly everything will be "significant". The per-sample → Fisher's method pipeline is the core of RIPPLE — don't skip it.

3. Not checking for contamination

Query-cell marker genes will always show strong "induced" gradients in nearby target cells due to ambient RNA, segmentation doublets, and transcript misassignment. Build a contamination gene list (known markers of your query type) and flag/exclude these from downstream interpretation.

4. Ignoring overdispersion

For most Xenium data, overdispersion is low (0.3–0.6). But for highly expressed genes, dispersion can exceed 2–3. If your median dispersion is consistently >2 across many genes, consider a quasi-Poisson or negative binomial GLM instead. RIPPLE's per-sample meta-analysis mitigates this by pooling independent estimates.

5. Too few replicates

Fisher's method with a sign consistency gate of 1.0 requires at least 3 replicates (and 4+ is much better). With N=2, the sign gate is trivially satisfied (both agree or both disagree), and Fisher's test has very low power. If N=1, you cannot use RIPPLE's meta-analysis — fall back to permutation testing within the single sample.

6. Confusing correlation with causation

A gradient shows spatial association, not causal direction. Gene X being induced near query cells could mean (a) query cells produce a signal that upregulates X, (b) target cells expressing X preferentially migrate toward query cells, or (c) both are attracted to the same niche. Use L-R databases (NicheNet, CellChat) and biological knowledge to distinguish these.
